# 对比框架

## 项目适用性

| 项目 | 同类程度 | 维护状态 | 最值得借鉴 | 不应照搬 |
|---|---|---|---|---|
| MacGesture | 最高：右键手势 + 轨迹层 | 核心代码 2023 后基本停更 | 遗留 down 处理、多屏、覆盖层真实坑 | 旧 Objective-C 全局状态、shielding level |
| LinearMouse | 高：CGEventTap + 手势按钮 | 2026 持续维护 | tap 健康检查、sleep/session stop-restart、纯状态协调器、测试隔离 | IOHID、设备管理、专用线程和整 App restart watchdog |
| Mac Mouse Fix | 中高：输入拦截 + 点击拖动 | 2026 持续维护 | 单线程状态所有权、失败重试、helper 诊断 | helper 双进程、滚轮/驱动复杂架构 |
| Hammerspoon | 中：通用 Event Tap / canvas | 2026 持续维护 | tap 真实状态、完整销毁重建、覆盖层所有权 | Lua 运行时和通用自动化框架 |

## 候选措施筛选

| 候选措施 | 证据强度 | 成本 | 回归风险 | 决策 |
|---|---:|---:|---:|---|
| 纯右键会话状态机 | 高 | 中 | 低，可测试 | 纳入，P0 |
| 所有异常路径统一 cancel | 高 | 低 | 低 | 纳入，P0 |
| mouse-up 丢失超时 | 中高 | 低 | 低 | 纳入，P0 |
| sleep/session inactive 时 stop，恢复时重建 | 高 | 中 | 低 | 纳入，P0 |
| 10 秒 tap health check | 高 | 低 | 低 | 纳入，P1 |
| 每 10 秒创建 test tap，失败就重启 App | 中 | 中 | 中高 | 不纳入 |
| 专用 Event Tap 后台线程 | 混合 | 高 | 高 | 不纳入 |
| 距离降采样 + 点数上限 + 增量长度 | 高 | 中 | 低 | 纳入，P0 |
| 按起点选屏 + screen change 通知 | 高 | 中 | 低 | 纳入，P0 |
| shielding level 改 statusBar | 中高 | 低 | 低 | 纳入，P1 |
| 真实 listener 状态展示 | 高 | 中 | 低 | 纳入，P1 |
| 普通右键保留 flags / clickState | 中 | 低 | 低 | 纳入，P1 |
| 立即重写 KeySender 为显式 flagsChanged | 混合 | 中 | 中 | 暂不纳入，仅验证 |
| 标准纯单元测试 | 高 | 中 | 低 | 纳入，P0 |
| 默认测试发送真实系统事件 | 反证强 | 中 | 高 | 禁止 |

| 自动更新、遥测、公证 | 与边界无关 | 高 | 中 | 不纳入 |

## 最小可靠架构

```text
AppDelegate
  ├─ 用户启停与真实状态显示
  ├─ sleep / wake / session / screen notifications
  └─ EventTap 生命周期协调

EventTap
  ├─ tap + runLoop source 的创建/销毁/健康检查
  ├─ 只把事件转换为 GestureSession 输入
  └─ 成功手势发快捷键；普通点击补发；异常取消不补发

GestureSession（纯逻辑，可测试）
  ├─ idle / tracking
  ├─ deferred click metadata
  ├─ bounded points / incremental length
  └─ begin / drag / finish / cancel / timeout

GestureOverlay
  ├─ 单窗口复用
  ├─ 当前屏幕坐标转换
  └─ statusBar level + screen change 清理
```
