# 资料来源

## L1：官方机制

1. [Apple CGEvent](https://developer.apple.com/documentation/coregraphics/cgevent)
   - Event Tap、事件字段和事件投递的官方入口。
2. [NSWorkspace.willSleepNotification](https://developer.apple.com/documentation/appkit/nsworkspace/willsleepnotification)
3. [NSWorkspace.didWakeNotification](https://developer.apple.com/documentation/appkit/nsworkspace/didwakenotification)
4. [NSWorkspace.sessionDidResignActiveNotification](https://developer.apple.com/documentation/appkit/nsworkspace/sessiondidresignactivenotification)
5. [NSWorkspace.sessionDidBecomeActiveNotification](https://developer.apple.com/documentation/appkit/nsworkspace/sessiondidbecomeactivenotification)
6. [NSApplication.didChangeScreenParametersNotification](https://developer.apple.com/documentation/appkit/nsapplication/didchangescreenparametersnotification)
7. [SMAppService](https://developer.apple.com/documentation/servicemanagement/smappservice)
   - 当前 SDK header 说明 `mainApp` 只负责后续登录时启动；LoginItem helper 非零退出才会被重新拉起。
8. 本机 macOS 15 SDK：`CoreGraphics.framework/Headers/CGEvent.h`
   - callback 在挂载的 CFRunLoop 上运行；tap 无响应或用户请求禁用时会收到 disabled 事件；可用 `CGEventTapEnable` 和 `CGEventTapIsEnabled` 管理。
9. 本机 `security help import`
   - `-A` 被明确标记为允许任何应用免提示访问私钥，且“不安全、不推荐”。
10. 本机 `launchd.plist(5)`
    - `KeepAlive=true` 会无条件保活；`SuccessfulExit=false` 或 `Crashed=true` 可只针对异常退出，但需要 LaunchAgent 架构。
