# SafariGestures 健壮性加固：问题拆解

## 问题类型

问题诊断 + 决策支持：识别当前实现的真实故障面，并决定哪些同类项目经验值得进入第 6 步，哪些会造成过度工程。

## 时效性

中等。Core Graphics / AppKit 的基础机制稳定，但 macOS 输入权限、睡眠唤醒和同类工具实现仍在变化。研究优先使用当前 macOS 15 SDK、2026 年仍维护的项目源码和近期 issue。

## 研究边界

研究对象：

- macOS 15+、Apple Silicon
- Swift 菜单栏 App
- `CGEventTap` 拦截 Safari 的右键 down / dragged / up
- 本机个人使用，不做 App Store 或第三方分发

不研究：

- 鼠标驱动、IOHID 设备适配
- 全系统滚轮平滑、按键映射
- 自动更新、遥测、商业分发

## 子问题

1. Event Tap 在超时、睡眠、用户切换、权限变化时如何失效，最低成本的恢复机制是什么？
2. 拦截 rightMouseDown 后，怎样保证任意中断路径都不会遗留 pressed 状态或幽灵点击？
3. 轨迹覆盖层在高轮询率、多屏、Mission Control 下有哪些已知坑？
4. 个人自用 App 为达到日常版标准，测试、签名、登录启动和进程拉起应做到哪一级？

## 纳入 Plan 的判断标准

每项建议按五个维度评估：

- 与当前代码是否直接对应
- 是否有官方机制或同类源码支持
- 是否有真实 issue / 修复记录
- 实现和维护成本
- 是否可自动或手动验证

只有“直接相关 + 有证据 + 成本合理 + 可验证”的措施进入第 6 步。
