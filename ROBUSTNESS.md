## 一、范围与原则

本轮只处理可靠性、安全性和可验证性，不增加新手势、不改变现有 8 个手势的操作方式。

实施原则：

1. 每类风险单独提交，避免多个核心行为混在一次改动里。
2. 先补可测试的状态机，再修改真实 Event Tap。
3. 自动测试通过后才做睡眠、多屏和真实右键测试。
4. `v0.2.0` 始终保留为可回退版本；不复制第二套工程。
5. 全部验收完成前，不替换 `~/Applications/` 中的日常版。
6. 普通单元测试不得向系统发送真实鼠标或键盘事件。

### B. 正式安装前的安全阻塞项

- [ ] B1. 收紧自签名私钥权限

旧版 `setup-signing-cert.sh` 使用 `security import -A`。系统帮助明确说明 `-A` 会允许任何应用免提示访问私钥，不推荐使用。该私钥又决定了辅助功能授权身份，因此必须在正式安装前处理。

目标方案：

- 不再使用 `-A`
- 私钥优先以不可导出方式导入
- 仅允许 `/usr/bin/codesign` 使用
- 验证重编后 Designated Requirement 稳定
- 处理已经用宽 ACL 导入的旧私钥；如需轮换，只做一次最终辅助功能重新授权

实施进度（当前改动）：

- [x] 本机 ACL 实测确认旧身份的签名权限为 `applications: <null>`，属于宽 ACL
- [x] 生成脚本移除 `-A`，改用 `-x` 和 `-T /usr/bin/codesign`
- [x] 增加新身份 ACL 检查与临时文件签名冒烟测试
- [x] 旧身份轮换改为显式 `--rotate-insecure-existing`，防止静默破坏现有授权
- [ ] 所有代码与真机验收完成后执行轮换，并完成最后一次辅助功能重新授权

## 三、已确认风险与处理要求

### A1. 右键状态机与异常复位

实施进度（`089d0c3` 及当前改动）：

- [x] 抽出不接触系统输入的 `GestureSession`
- [x] 新 down、tap disabled、Safari 失焦、停用统一走取消入口
- [x] 增加 5 秒 mouse-up 看门狗
- [x] 异常取消后进入 `discardingUntilMouseUp`，持续吞掉孤立 drag/up；新 down 可恢复
- [ ] 完成对应真机异常场景验收

当前风险：

- Event Tap 被禁用时只重新启用，没有清除 `isTracking` 和覆盖层
- Safari 在按住右键期间失去前台时，可能放行没有配对 `down` 的 `drag/up`
- mouse-up 丢失时没有超时兜底
- 停用或退出发生在手势中途时，没有统一的取消路径

实现要求：

- 抽出明确的 `idle / tracking / discardingUntilMouseUp` 会话状态
- 普通点击采用 deferred click：按下时暂存，释放时根据是否命中手势决定补发或丢弃
- 所有异常路径只走一个 `cancelTracking(reason:)` 入口
- tap 禁用、Safari 失焦、停用、退出、睡眠、用户会话失活、屏幕重配置时立即清轨迹并隐藏覆盖层
- 异常取消不得误触发快捷键，也不得弹出幽灵右键菜单
- 增加约 5 秒的会话看门狗，mouse-up 丢失后自动复位
- 收到新的 rightMouseDown 时若旧会话仍未结束，先取消旧会话再开始新会话

验收标准：

- [ ] 按住右键后 Cmd-Tab，松开时其他 App 不出现异常菜单或卡键
- [ ] 手势中途停用监听，轨迹立即消失，重新启用后正常
- [ ] 模拟 tap disabled 后不会执行旧手势
- [ ] 丢失 mouse-up 后最多 5 秒停止 tracking；延迟 up 被吞掉，或下一次新 down 可恢复

- [x] 睡眠、唤醒和用户会话切换使用完整 stop/start
- [ ] 完成睡眠唤醒和用户切换真机验收

### A3. 轨迹采样与资源上限

实施进度（当前改动）：

- [x] 会话按约 2px 采样，识别点最多 512 个并在上限处降采样
- [x] 覆盖层独立限制为最多 512 个绘制点
- [x] 覆盖层路径长度改为增量累计，不再每次全量遍历
- [x] 会话结束后释放轨迹数组容量
- [ ] 完成 60 秒高频移动与内存回落真机验收

当前风险：每个 dragged 事件都追加点；覆盖层每次追加都重新遍历全部轨迹计算长度，长按和高轮询率鼠标下会形成 O(n²) 开销。

实现要求：

- 相邻点距离不足约 2px 时不追加
- 单次会话设置明确上限，建议最多 512 个展示/识别点
- 路径长度改为增量累计，不再每次遍历全部点
- 到达点数上限后采用降采样或保留关键转折，不允许数组无限增长
- 异常大容量会话结束后释放高水位容量，避免常驻内存永久抬高

验收标准：

- [ ] 按住右键持续移动 60 秒，点数不超过上限
- [ ] 期间 Event Tap 不超时，轨迹仍跟手
- [ ] 会话结束后内存回落且不持续增长

## 七、明确暂缓

以下项目本轮不做，除非实测证明必要：

- 专用后台 Event Tap 线程：同类项目出现过线程切换导致竞态和手势卡死；当前逻辑保持主 RunLoop 更简单
- 无条件 `KeepAlive`：会导致用户正常退出后 App 又被拉起
- 定时创建 test tap 并失败后重启整个 App：当前只需检查现有 tap 并在进程内重建
- 未复现前重写 KeySender：先验证 modifier flags，避免改变已通过真机测试的快捷键发送行为
- 自动更新、联网崩溃上报、遥测
- Developer ID、公证和面向他人分发

如果后续确实观察到进程异常退出，再单独评估只针对异常退出的 LaunchAgent；不与本轮核心手势改动混合。
